If else and Switch

If else Syntax

if(condition){
   //Block of code if condition is true
}
else{
   //Block of code if condition is false.
}

Example

let age = 18;
if (age >= 18){
   console.log("You are eligible to vote")   //This gets printed
else{
   console.log("Sorry you cannot vote.")
  }
}

Switch case Syntax

switch(condition){
  case identifier1:
  //block of code
  break;

  case identifier2:
  //block of code
  break;

default:
  //block of code
  break;
}

Example

let age = 12;
switch(age){
   case 18:
   console.log("you can vote");
  break;

  case 12:
  console.log("you cannot vote");   //this is printed
  break;

  default:
  console.log("some error occured");
  break;
}

Difference between if else and switch case

If else is used to evaluate character, integers, pointer, floating-point type and boolean type.
While switch is used to evaluate character expressions and integer.

If else tests for both logical expressions and equality.
While switch tests for equality.

Switch statements are faster in compare to if-else statements.


Let's observe a difference between "==" and "===":

"==" operator performs type coercion (automatic conversion from one data type to another) and converts string to number before comparing.
"=== (strict equality operator) " operator doesn't perform type coercion and directly compares number with a string.

let age = "18";   //string
if (age == 18){   //Comparing with integer
   console.log("you can vote");
}
else{
   console.log("You cannot vote");
}

In above case, the output is "you can vote". The reason behind it is (==) operator only checks for the value ignoring the type of data it is comparing.

Now let's observe this.

let age = "18";   //string
if (age === 18){   //Comparing with integer
   console.log("you can vote");
}
else{
   console.log("You cannot vote");
}

Unlike (==) operator (===) checks both the data type as well as the value compared.

Since we are comparing integer with string, the condition is skipped to else block and the output is "you cannot vote".
