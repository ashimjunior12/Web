var, let and const

  1. var:-
    var is globally scoped or function/locally scoped. It's value can be accessed from everywhere.
    But when it is declared within a function, it's value can only be accessed within that function.

    var string = "this is a string";
    //It's value can be accessed and updated from anywhere.

    var string = "hello";
    function newFunction() {
          var hello = "hello";
    }
    console.log(hello);    error: hello is not defined.

    var can be re-declared and updated.

    var string = "this is a string";
    string = "this is another one"    //value of string gets updated
    var string = "this is a new string"  //string can be re-declared, which alredy existed

    var variables are hoisted to the top of their scope and initialized with a value of undefined.

    console.log(name);
    var name = "My name is ram";

    is interpreted as:-

    var name;
    console.log(name);
    var name = "My name is ram";
    console.log(name);      //undefined

    Problem with var:-

    var name = "my name is ram";
    var times = 3;
    if(times < 4){
        var name = "my name is hari";
    }
    console.log(name);  //"my name is hari"

    if(true){
    console.log(name);   //my name is hari
    }

    With the above code, the problem is, whenever we want the name, value of name is updated as "my name is hari", everywhere.


  2. let:-
    let is block scoped. Its value can be accessed within its block only.

    let name = "my name is ram";
    if(true){
      let greet = "hello";
    }
    console.log(greet);   //greet is not defined.

    let can be updated but can't be redeclared(within its scope).

    let name = "my name is ram";
    name = "my name is hari";   //value of name is updated
    console.log(name);    //my name is hari

    let name = "my name is ram";
    let name = "my name is hari";
    console.log(name);   //error cause name has already been declared

    but same varibale can be defined in different scopes

    let name = "my name is ram";
    if(true){
       let name = "my name is hari";
       console.log(name)   //my name is hari
    }
    console.log(name);   //my name is ram

    Hoisting of let:-
    let are hoisted at top, however let keyword is not initialized.

    console.log(greeting);    //Refrence Error
    greetng = "hello";

  3. const:-
    variable declared with const keyword maintain constant value throughout the entire program.
    const are block scoped.

    const can neither be updated nor be re-declared.

    const greeting = "hello";
    greeting = "hi"     //greeting cannot be updated

    const greeting = "hello";
    const greeting = "hi"    // greeting cannot be re-delcared.

    properties of const object can be updated.

    const greeting{
         message = "hello";
         times = 4;
    }

    we can do:-

    greeting.message = "say hi";   //This will update the value message of greeting.

    Hoisting of const:-
    Just like var and let, const is hoisted at the top but its value is not initialized.
